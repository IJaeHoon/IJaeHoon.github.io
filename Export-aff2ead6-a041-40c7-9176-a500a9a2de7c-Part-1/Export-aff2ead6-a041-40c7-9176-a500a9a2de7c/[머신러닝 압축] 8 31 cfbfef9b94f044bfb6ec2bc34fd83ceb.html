# 회귀는 아무것도 없고 분류는 소프트맥스를 건다 출력일 경우에만

근데

히든레이어에도 액티베이션 펑션을 쓴다 ⇒ 성능에 지대한 영향을 미침 ⇒ 히든레이어에모든것들은회귀처럼동작한다, ‘근데’ 다이렉트펑션.. 레이어를 늘인다고 성능이 별로 높아지지않아,,

단순 회귀 반복만으로는 그다지 성능이 늘지않는다 ← 마빈 민스키

단순출력은 끝이 일차방정식 , 뻘짓하게된거 수없이 많은 레이어를 건너왔는데, 다이렉트 펑션대로 구해지면

레이어 한 개 있는거랑 다를게 없다 같은 원리 → 일차 방정식을 복잡한 형태로 쓴 것일 뿐이다.

‘그래서’

출력을 비선형으로 만들어버린 것, 그래야 각 특색을 가진 조합이 다시 조합돼서

딥러닝의 히든레이어에는 비선형 함수를 써서 각자 역할을 할 수 있게 해주는 것.

‘시그모이드’ ← 활성화함수 3번 미분을 하면 값이 0에 가까워진다. 시그모이드 = 0~1사이를 뱉는데 3번 뱉으면 0에 가까워짐..

활성화함수를 미분하고 미분하고 미분하고하니, 0에 가까워지고, 레이어가 깊어지면 시그모이드 함수는 써먹을수 없다

그래서 다른 활성화 함수를 찾았는데, 그게

‘렐루’ 함수

렐루함수 찾아낸게 재프리 힌튼

‘활성화함수는 반드시 미분이 가능해야한다’ 를 ≠ 깨버린 것,

if x가 0일때만 미분하지마라 조건만 걸면 해결된것,

-활성화 함수 거는 것-

히든레이어 → 액티베이션 펑션 ⇒ ‘렐루’ (지금은 조금 변형된 리키 렐루, Ricky ReLU를 사용한다함)

출력레이어 → 액티베이션 펑션 ⇒ ‘회귀냐 분류에 따라서’ 달라

‘딥러닝 모델’ 이런식으로 내가 모델만들어서 써먹을 일은 거의 없을 것, 코드를 알아야하니까 써보는거지,

‘프리트레이드 모델’을 쓴다 메타나 구글 등이 만든 모델 쓰고 거기서 훈련만 하는거

열심히 배우곤 있지만, 백그라운드 날리지지, 모델을 만들 일은 없다, 구글 브레인, 마이크로소프트 애저팀, 인텔 오픈비노라이브러리팀에 들어가지 않는 이상

우리가 람마 알파카 지피티 만들 일 없을것, 만들려해도 단기간에 지피티 나올 일은 없을 것, 지피티가 프리트레인드 모델 / 제너레이티브 프리트레인드 트랜스포머

지피티를 쓴다 → 지피티에서 트레이닝 다 시켜놓을테니까

대한민국 할아버지들의 대화패턴학습 → 지피티 섞어서 출시하는거

우리가 언제 지피티를 만들거야 ← 어차피 남이 만든거 땡겨서 써야돼

★ 회사 취업이나 일반적인 상황에선 ‘프리트레인드’ 모델을 땡겨와서 쓰는게 역량이 될 것,

실무에서 필요한 능력, 지금 짜본건 허깅페이스를 쓰기위해서 이해하기 위해서 백그라운드로 공부하는것, 실무에선 저렇게 쌓아서 만들 일은 많지는 않다
